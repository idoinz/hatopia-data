# Hatopia website data

JSON and images for the Hatopia site. Use this as a **separate** GitHub repo so you can update content without redeploying the main site.

## Contents

- **info.json** – Daily date, Roaming Oak, Flawless Flouride, and notes (Daily / Weekly / Seasonal / Other)
- **manifest.json** – Image lists for Flowers and Info sections (generated by the Python script or by hand)
- **flowers.json** – Flower image filenames (fallback when manifest is not used)
- **animals.json** – Animals list (name, emoji, foods)
- **images/** – Image files used by the site
  - **flowers/** – Flower images
  - **info/Daily/, info/Weekly/, info/Seasonal/, info/Other/** – Info tab images
  - **groups/** – Group icons (e.g. SEA.png)
  - **hatopia.png** – Logo (at root of images)

## Using this repo

1. Push this folder to a new GitHub repo (e.g. `yourusername/hatopia-data`).
2. In your main Hatopia site code, point `fetch()` URLs to the raw GitHub URLs, for example:
   - `https://raw.githubusercontent.com/YOUR_USERNAME/hatopia-data/main/info.json`
   - `https://raw.githubusercontent.com/YOUR_USERNAME/hatopia-data/main/manifest.json`
   - Same pattern for flowers.json, animals.json, and image paths under `images/`.
3. When you edit files in this repo (on GitHub or by push), the live site will load the new data on refresh—no Netlify redeploy needed.

## Updating data

- Edit JSON files here (or use the Hatopia Python script and copy the updated files into this folder, then commit and push).
- Add or replace images in the appropriate `images/` subfolders, then update `manifest.json` or `flowers.json` if needed.
